Storybook has a powerful feature set that supports all major JavaScript frameworks. But this also makes it tricky to optimize.</p><p>In mature open source projects, bloat is bigger than any one issue. This post shares how we overhauled Storybook 9’s bundle to deliver faster installs, smaller lockfiles, and fewer upgrade headaches.</p><ul><li>🗜️&nbsp;Cut install size by over 50%</li><li>🧹&nbsp;Overhaul internal dependency structure</li><li>📦&nbsp;Move key addons into core</li><li>🪶&nbsp;Support dev-only “minimal” install</li></ul><p><em>Special thanks to </em><a href="https://e18e.dev/?ref=storybookblog.ghost.io"><em>e18e Ecosystem Performance</em></a><em> for supercharging our efforts.</em></p><hr /><p>Prefer video? We held a live session discussing this topic.</p><figure class="kg-card kg-embed-card"></figure><hr /><h2 id="the-bloat-problem">The bloat problem</h2><p>There is no silver bullet to bloat. It can come in many sizes and shapes. Massive <code>node_modules</code>. Unwanted transitive dependencies. Conflicts between project deps and Storybook. Noisy CLI output. And more.</p><p>Bloat is all of these things. Worse, developers came to expect Storybook to be heavy. For Storybook 9, we needed a holistic solution to shatter preexisting expectations.</p><h2 id="how-we-tackled-bloat">How we tackled bloat</h2><p>Storybook 9 is based on the following principles:</p><ul><li>Fewer packages</li><li>Replace heavy dependencies</li><li>Prebundle everything</li><li>Only install what you need</li></ul><h3 id="fewer-packages">Fewer packages</h3><p>Storybook 9 includes major improvements to our package structure. Package boundaries enforce modularity, but they also lead to various kinds of bloat, such as duplicated dependencies and more dependencies in the user’s <code>package.json</code>. To fix this, we consolidated and reduced the number of Storybook packages.</p><p>Now to minimally use Storybook, you need a single core package, <code>storybook</code>, and a framework package <code>@storybook/nextjs-vite</code> , <code>@storybook/angular</code>, <code>@storybook/sveltekit</code>, etc.</p><p>Functionality that previously existed in addons—like Controls for adjusting component props—was consolidated into Storybook's core. Other functionality, like MDX and autogenerated component docs, was consolidated to a single package, <code>@storybook/addon-docs</code>. All packages contain a peer dependency on <code>storybook</code>.</p><figure class="kg-card kg-code-card"><pre><code class="language-diff">// package.json
{
  @storybook/addon-docs
-  @storybook/addon-essentials
-  @storybook/addon-interactions
  @storybook/addon-onboarding
-  @storybook/blocks
-  @storybook/react
  @storybook/react-vite
-  @storybook/test
+  @storybook/addon-vitest
  eslint-plugin-storybook
  storybook
}</code></pre><figcaption><p><span style="white-space: pre-wrap;">The difference between necessary Storybook 8 and 9 packages in package.json for a typical project</span></p></figcaption></figure><p>This simplified package structure prevents dependency hell when upgrading Storybook. It also allows us to prebundle packages, which is the next step in our approach.</p><h3 id="prebundle-aggressively">Prebundle aggressively</h3><p>Storybook aggressively prebundles dependencies because JS package managers can mess things up: different versions, different installs, lockfiles, unexpected conflicts.</p><p>What’s prebundling? It means shipping known-good versions of key packages inside the core <code>storybook</code> package, so you don’t have to resolve them yourself at install time. This dramatically reduces install errors, mismatches, and dependency conflicts. It also flattens the dependency tree and tree-shakes away the parts of a package you don’t use, which often leads to dramatic size reduction.</p><figure class="kg-card kg-image-card kg-width-wide"><img alt="Storybook bloat? Fixed." class="kg-image" height="1120" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2025/07/Storybook-Bloat-Fix-Feedback.png" width="2000" /></figure><h3 id="find-and-replace-heavy-dependencies">Find and replace heavy dependencies</h3><p>While fixing Storybook’s package structure was a step in the right direction, we also swapped in lightweight alternatives for our own dependencies wherever possible. <a href="https://e18e.dev/?ref=storybookblog.ghost.io">e18e</a>’s “<a href="https://e18e.dev/guide/cleanup.html?ref=storybookblog.ghost.io">cleanup</a>” resource helped identify optimization opportunities and recommended smaller packages. This reduced our bundle size by tens of megabytes.</p><p>For example, using <a href="https://www.npmjs.com/package/polka?ref=storybookblog.ghost.io">Polka</a> (67KB) instead of <a href="https://www.npmjs.com/package/express?ref=storybookblog.ghost.io">Express</a> (2.2MB) trimmed 2MB instantly and delivered faster end user performance.</p><figure class="kg-card kg-gallery-card kg-width-wide kg-card-hascaption"><div class="kg-gallery-container"><div class="kg-gallery-row"><div class="kg-gallery-image"><img alt="Storybook bloat? Fixed." height="1455" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2025/07/Storybook-Bloat-Fix-Feedback--1-.png" width="2000" /></div><div class="kg-gallery-image"><img alt="Storybook bloat? Fixed." height="985" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2025/07/Storybook-Bloat-Fix-Feedback--2--1.png" width="2000" /></div></div></div><figcaption><p><span style="white-space: pre-wrap;">Left: Express (2.2MB). Right: Polka (67KB)</span></p></figcaption></figure><h3 id="only-install-what-you-need-minimal-%E2%80%9Cdev-only%E2%80%9D-storybook">Only install what you need: Minimal “dev-only” Storybook</h3><p>If you only want to write stories without features like automated docs and component testing, we added an option to install a minimal, dev-only Storybook. This allows you to further cut install size and <code>package.json</code> dependencies.</p><figure class="kg-card kg-image-card"><img alt="Storybook bloat? Fixed." class="kg-image" height="704" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2025/07/create-storybook-1.png" width="1666" /></figure><h2 id="try-storybook-9-today">Try Storybook 9 today</h2><p>We’re committed to making Storybook the fastest, leanest frontend workshop in the world. As maintainers, we continually balance stability, performance, and keeping up with the broader JavaScript ecosystem.</p><p>Storybook 9 introduces incredible new component testing features while cutting bundle size in half. Give it a try in a new project:</p><pre><code class="language-bash">npm create storybook@latest
</code></pre><p>If you have an existing Storybook project, first upgrade it to Storybook 9. Use our automated migration wizard to help you along the way:</p><pre><code class="language-bash">npx storybook@latest upgrade
</code></pre><p>We also created a&nbsp;<a href="https://storybook.js.org/docs/9/migration-guide?ref=storybookblog.ghost.io">migration guide</a>&nbsp;to help fill in the gaps.</p><figure class="kg-card kg-embed-card"><blockquote class="bluesky-embed"><p lang="en">We've added a lot of features, and a bit of bloat, over the years.
