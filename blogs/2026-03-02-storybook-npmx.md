<img alt="Storybook 💙 npmx" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2026/03/section.jpg" /><p>Today, <a href="https://npmx.dev/blog/alpha-release?ref=storybookblog.ghost.io">npmx launches its alpha release</a>. We’re thrilled to be part of the moment and want to celebrate npmx: a radically better npm browsing experience that surfaces the information you need to help choose the best packages. We're also excited to share our small contribution to the mix: adding live UI component examples so you can see what you'll get before installing.</p><h2 id="what-is-npmx">What is npmx?</h2><p>If you’ve ever tried to evaluate an npm package, you know the drill. Open <a href="http://npmjs.com/?ref=storybookblog.ghost.io">npmjs.com</a>, skim the README, then bounce to GitHub for the source, <a href="https://pkg-size.dev/?ref=storybookblog.ghost.io">pkg-size.dev</a> for the size, Are The Types Wrong for the TypeScript story, and maybe Socket.dev for a security check. You’re six tabs deep before you’ve even decided if the package is worth installing.</p><p><a href="https://npmx.dev/?ref=storybookblog.ghost.io">npmx.dev</a> is changing that. It’s a fast, modern, community-built browser for the npm registry that puts everything you need in one place: install size, module format badges, outdated dependency warnings, vulnerability metadata, download trends, and more. And it does all this with dark mode, keyboard navigation, a fantastic code viewer, and much more. This is just the beginning!</p><figure class="kg-card kg-image-card"><img alt="Storybook 💙 npmx" class="kg-image" height="1069" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2026/03/image-1-1.png" width="1150" /></figure><p>We’ve been watching npmx closely. Today we’re excited to announce that Storybook and Chromatic are collaborating with npmx to bring component stories directly into the npm browsing experience. And as it turns out, the npmx team has also adopted Storybook for their own development. A natural fit for a project moving this fast.</p><h2 id="why-we-care">Why we care</h2><p>As maintainers of high-profile npm packages, we love the JS ecosystem. npm's package registry sits at the core of the ecosystem and has helped it thrive over the past decade.</p><p>But with this success comes lots of problems: security supply chain attacks, package bloat, a sea of stale and/or unmaintained packages, outdated dependencies, format changes, runtime compatibility, and so on.</p><p>Every package maintainer and consumer needs to contend with these issues. The community has built a variety of tools and databases to help keep your packages and dependencies healthy, but you have to go find the information for yourself. With npmx, all of this information is clearly visible in the package page.</p><p>Increased transparency helps you pick the best package for the job. It also compels package maintainers to level up, and also helps them do it. We think npmx is going to be the best thing to happen to the JS ecosystem since Vite.</p><h2 id="bringing-stories-to-the-package-page">Bringing stories to the package page</h2><p>Many of the most popular design systems and UI libraries in the world are built and documented with Storybook. We’re bringing this information to npmx so you can easily learn about and try a UI library before you install it.</p><p>To start with, we’ve settled on a simple solution of showing a link to the package’s Storybook in the sidebar of npmx, making it easy for users to view and interact with the components.</p><figure class="kg-card kg-image-card kg-width-wide"><img alt="Storybook 💙 npmx" class="kg-image" height="222" src="https://storage.ghost.io/c/f9/d7/f9d78a80-56f8-4adc-8ba1-0e2d6ee72b16/content/images/2026/03/image.png" width="1135" /></figure><p>We’re iterating with the npmx team to figure out the best way to surface stories alongside the rest of the package metadata. But the vision is clear: use live stories as an additional way to evaluate component libraries, right next to the other crucial information like install size and vulnerabilities.</p><p>To surface your library’s components in npmx, <a href="https://storybook.js.org/docs/sharing/package-composition?ref=storybookblog.ghost.io#for-authors">just set a <code>storybook.url</code> field in your <code>package.json</code></a>, pointing to your deployed Storybook. Popular UI libraries like <a href="https://storybook.chakra-ui.com/?ref=storybookblog.ghost.io">Chakra UI</a> and <a href="https://react-aria.adobe.com/react-aria-starter/?path=/docs/breadcrumbs--docs&amp;ref=storybookblog.ghost.io">React Aria</a> already do this, and we hope that this npmx collaboration will make the practice widespread.</p><h2 id="npmx-adopts-storybook">npmx adopts Storybook</h2><p>We’re also helping npmx adopt Storybook for their own development workflow.</p><p>The npmx community <a href="https://github.com/npmx-dev/npmx.dev/pull/1270?ref=storybookblog.ghost.io">set up Storybook</a> for their own Nuxt project, with Storybook core team members pitching in on the PR. The community response was immediate and enthusiastic.</p><p>Additionally, <strong>Chromatic is sponsoring the npmx project</strong> with free <a href="https://www.chromatic.com/features/visual-test?ref=storybookblog.ghost.io">visual</a> and <a href="https://www.chromatic.com/features/accessibility-test?ref=storybookblog.ghost.io">accessibility snapshot testing</a>. This means every PR to npmx is automatically checked for visual regressions and WCAG compliance — the same workflow used by the world’s top UI teams. Now available to the npmx community at no cost.</p><h2 id="the-npmx-vibes">The npmx vibes</h2><p>npmx was started by <a href="https://bsky.app/profile/danielroe.dev?ref=storybookblog.ghost.io">Daniel Roe</a> and <a href="https://bsky.app/profile/patak.dev?ref=storybookblog.ghost.io">Matias Capeletto (patak)</a>, names you might recognize from the Nuxt and Vite ecosystems. In just a few months, it’s grown to 150+ contributors and 2,000+ GitHub stars, fully open source with a <a href="https://github.com/npmx-dev/npmx.dev/blob/main/GOVERNANCE.md?ref=storybookblog.ghost.io">clear governance model</a> that welcomes all kinds of contributions.</p><p>What stands out is the <em>culture</em>. People are opening their first-ever OSS PRs on npmx. Contributors are saying the project reignited their love for the web. The community recently took an entire week off together, closing Discord and pausing GitHub to touch grass. Anthony Fu <a href="https://bsky.app/profile/antfu.me/post/3meupr6m3pc2g?ref=storybookblog.ghost.io">called it out</a>: <em>“Simply, more OSS projects should DO THIS.”</em></p><p>And today, on launch day, that energy is on full display: dozens of community members, projects, and collaborators are all publishing their own announcements and celebrations of npmx at the same time. This isn’t a corporate marketing campaign. It’s a community showing up for something they genuinely believe in.</p><p>We’re huge fans of what the npmx community is building. A faster, more informative, more human npm browsing experience is something the entire JavaScript ecosystem benefits from. Today’s alpha is just the starting line, and we’re proud to be running alongside them. 💙</p><hr /><p><em>npmx launches today! Check out </em><a href="https://npmx.dev/?ref=storybookblog.ghost.io"><em>npmx.dev</em></a><em>, join the community at </em><a href="https://chat.npmx.dev/?ref=storybookblog.ghost.io"><em>chat.npmx.dev</em></a><em>, and follow along on </em><a href="https://bsky.app/profile/npmx.dev?ref=storybookblog.ghost.io"><em>Bluesky</em></a><em>. See what others are saying about the launch across the ecosystem.</em></p><p><em>To add your Storybook to your package on npmx, add a </em><a href="https://storybook.js.org/docs/sharing/package-composition?ref=storybookblog.ghost.io#for-authors"><em><code>"storybook"</code> field</em></a><em> to your <code>package.json</code>:</em></p><pre><code class="language-json">{
  "storybook": {
    "url": "&lt;https://your-storybook-url.com&gt;"
  }
}</code></pre>
